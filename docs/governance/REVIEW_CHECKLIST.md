# Review Checklist

本文档定义每次 Review 必须检查的问题。适用于数据、Feature、模型、策略和回测。

## 数据

- 是否存在缺失？
- 是否存在重复？
- 是否存在未来数据？
- Train/Test 是否严格按时间？
- 时间间隔是否连续？
- OHLC 是否存在逻辑错误？
- 成交量是否存在异常值？

## Feature

- 是否解释一种市场现象？
- 是否可能造成 Leakage？
- 是否只使用当前时刻及过去数据？
- 是否重复表达已有 Feature？
- 是否有明确 Research Hypothesis？
- 是否有明确 Calculation Method？
- 是否有明确 Expected Effect？
- 是否记录 Potential Risks？

## 模型

- 是否建立 Baseline？
- 是否出现过拟合？
- Train/Test 切分是否严格按时间？
- 重要 Feature 是否符合市场认知？
- 模型概率分布是否合理？
- Precision、LogLoss、AUC 是否同时查看？
- 是否分析了失败案例？

## 策略

- 是否考虑手续费？
- 是否考虑滑点？
- 是否考虑持仓？
- 是否真正盈利？
- 是否定义开仓、平仓和空仓规则？
- 是否定义止盈止损？
- 是否在不同市场状态下检查稳定性？
- 是否确认交易结果不是由未来函数造成？

## 结论

Review 不能只看指标是否提高。

每次 Review 必须回答：

1. 这个结果是否可信？
2. 这个结果是否可复现？
3. 这个结果是否有市场解释？
4. 这个结果是否能转化为交易收益？
5. 下一步应该继续、修改、废弃，还是进入回测？
