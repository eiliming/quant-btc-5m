# Research Workflow

本文档定义本项目的标准研究流程。该流程用于保证研究先于实现，决策可追踪，实验可复盘。

## 标准流程

```text
市场观察
↓
提出市场假设
↓
设计 Feature
↓
Review Feature
↓
Codex 实现
↓
训练 Baseline
↓
分析模型
↓
分析失败案例
↓
优化 Feature
↓
重新训练
↓
策略设计
↓
回测
↓
总结
```

## 阶段说明

### 市场观察

研究者观察 BTCUSDT 5分钟市场中的价格、波动、成交量、趋势、横盘、突破、回撤和异常行为。

### 提出市场假设

研究者将观察转化为可验证假设。例如：某种波动收缩后的突破行为是否提高下一根 K 线上涨概率。

### 设计 Feature

研究者根据假设设计 Feature。每个 Feature 必须对应明确的市场现象，并记录在 Feature 文档中。

### Review Feature

在实现之前检查 Feature 是否存在未来数据、是否重复表达已有信息、是否有明确预期作用、是否可能造成 Leakage。

### Codex 实现

Codex 根据明确的 Feature 设计、实验设计或工程需求进行实现。

Codex 不能参与研究决策，不能替代研究者决定市场假设、Feature 是否有意义、策略是否应当采用。

### 训练 Baseline

使用简单、可解释、可复现的模型建立基准结果，为后续优化提供参照。

### 分析模型

分析模型指标、概率分布、重要 Feature、稳定性、不同市场状态下的表现。

### 分析失败案例

重点分析模型错误、低置信度、高置信度错误、特定市场状态失效等问题。

### 优化 Feature

根据失败案例和市场假设调整 Feature，而不是盲目堆叠更多特征。

### 重新训练

在相同实验规范下重新训练，记录版本、参数、指标和结论。

### 策略设计

策略层根据模型概率、交易成本、滑点、持仓规则、风控规则决定交易行为。

### 回测

严格按时间顺序回测，禁止随机打乱时间序列，禁止未来函数，并纳入手续费、滑点和持仓约束。

### 总结

每个 Phase 结束后，必须记录研究日志，总结关键决策、有效发现、失败原因、经验教训和下一步。

## 角色边界

研究者负责市场理解、研究假设、Feature 设计、实验目标、模型分析、策略设计和最终决策。

Codex 负责实现明确需求、维护工程结构、编写测试、整理文档和执行可验证任务。

Codex 不能参与研究决策。
